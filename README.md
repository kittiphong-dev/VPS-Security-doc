# คู่มือพื้นฐาน VPS Security (ความปลอดภัยบน Server ที่ Dev ควรรู้!)

คู่มือฉบับนี้สรุปเนื้อหาและคำสั่งทีละขั้นตอน (Step-by-Step) จากวิดีโอช่อง MilerDev (EP.6) สำหรับผู้เริ่มต้นใช้งาน VPS (แนะนำระบบปฏิบัติการ Ubuntu 22.04 LTS) เพื่อช่วยลดความเสี่ยงจากการถูกโจมตีตั้งแต่แรกเริ่มติดตั้งระบบ

---

## ⚠️ กฎเหล็กสำคัญก่อนเริ่มทำการตั้งค่า (Pre-flight Check)

> [!IMPORTANT]  
> **ทำ Snapshot Backup เสมอ:** ก่อนเริ่มแตะต้องไฟล์ Configuration สำคัญใดๆ (เช่น SSH, Firewall) ให้เข้าไปที่หน้าจัดการ VPS ของผู้ให้บริการ แล้วสร้าง Snapshot เอาไว้ก่อน เพื่อใช้เป็นจุดย้อนกลับหากตั้งค่าผิดพลาดจนล็อกอินไม่ได้

> [!WARNING]  
> **อย่าเพิ่งปิด Terminal เดิม:** ระหว่างที่ทำการทดสอบตั้งค่าใหม่ ห้ามปิดหน้าต่าง Terminal ที่กำลังเชื่อมต่ออยู่ปัจจุบัน จนกว่าจะเปิด Terminal อันใหม่แล้วทดสอบล็อกอินด้วยสิทธิ์ใหม่ผ่านได้ 100%

---

## 🛠️ ขั้นตอนการติดตั้งและตั้งค่าระบบทีละสเต็ป (Step-by-Step)

### Step 1: อัปเดตระบบและติดตั้งเครื่องมือพื้นฐาน
อัปเดตระบบปฏิบัติการให้เป็นเวอร์ชันล่าสุดเพื่ออุดรอยรั่วความปลอดภัย และติดตั้งเครื่องมือที่จำเป็น
* **ทำที่ไหน:** Web Terminal ของผู้ให้บริการ VPS (ล็อกอินครั้งแรกด้วยสิทธิ์ `root`)

```bash
# 1. อัปเดตรายชื่อแพ็กเกจใหม่ล่าสุด
apt update

# 2. ทำการอัปเกรดระบบ (ตอบตกลงอัตโนมัติด้วย -y)
apt upgrade -y

# 3. ติดตั้งเครื่องมือพื้นฐานและแพ็กเกจความปลอดภัย
# (curl, wget, ufw, fail2ban, unattended-upgrades)
apt install -y curl wget ufw fail2ban unattended-upgrades
```

---

### Step 2: สร้าง User ใหม่เพื่อความปลอดภัย (แทนการใช้ root)
เราไม่ควรใช้สิทธิ์สูงสุดอย่าง `root` ในการทำคำสั่งทั่วไป เพราะหากพิมพ์ผิดพลาดเพียงนิดเดียวอาจทำให้ระบบเสียหายทั้งหมด
* **ทำที่ไหน:** Web Terminal (สิทธิ์ `root` ปัจจุบัน)

```bash
# 1. สร้างผู้ใช้ใหม่ (เปลี่ยนคำว่า 'miler' เป็นชื่อผู้ใช้ที่คุณต้องการ)
adduser miler
```
> [!NOTE]  
> ระบบจะให้ตั้งรหัสผ่านใหม่ และกรอกข้อมูลทั่วไป เช่น ชื่อ-นามสกุล ส่วนข้อมูลเสริมอื่นสามารถกด Enter ข้ามไปได้จนจบ

```bash
# 2. เพิ่มผู้ใช้ใหม่เข้าไปในกลุ่ม sudo เพื่อให้สามารถใช้สิทธิ์ผู้ดูแลระบบได้
usermod -aG sudo miler

# 3. ทดสอบการสลับหน้าผู้ใช้ใหม่และลองรัน sudo เพื่อเช็คความถูกต้อง
su - miler
sudo whoami
```
> [!NOTE]  
> หากระบบเรียกหารหัสผ่านและแสดงคำว่า `root` แปลว่าตั้งค่าเสร็จสิ้นถูกต้อง จากนั้นให้พิมพ์คำว่า `exit` เพื่อกลับไปยังสิทธิ์ `root`

---

### Step 3: การสร้างและใช้งาน SSH Key แทนรหัสผ่าน
เพิ่มระดับความปลอดภัยขั้นสูงด้วยการใช้กุญแจดิจิทัลในการล็อกอินแทนการคีย์พาสเวิร์ดผ่านคีย์บอร์ด
* **ทำที่ไหน:** ทำสลับกันระหว่าง Terminal/PowerShell บนเครื่องคอมพิวเตอร์ของคุณเอง และ บน Server

#### ทางเลือกที่ 1: ตั้งค่าอัตโนมัติ (แนะนำสำหรับ macOS / Linux)
หากคุณใช้ macOS คุณสามารถใช้เครื่องมือ `ssh-copy-id` เพื่อส่ง Public Key ไปยัง Server ได้โดยอัตโนมัติด้วยคำสั่งเดียว:

1. **[บน Mac ของคุณ]** เปิด Terminal และสร้างคู่กุญแจ:
   ```bash
   ssh-keygen -t ed25519 -C "Miler-VPS"
   ```
2. **[บน Mac ของคุณ]** ส่ง Public Key ไปที่ Server ด้วยคำสั่ง (ระบบจะถามรหัสผ่านของผู้ใช้ `miler` บน Server):
   ```bash
   ssh-copy-id miler@<IP_Serverของคุณ>
   ```

---

#### ทางเลือกที่ 2: ตั้งค่าด้วยตนเอง (Manual - สำหรับ Windows หรือกรณีทั่วไป)
1. **[บนคอมพิวเตอร์ส่วนตัว]** เปิด PowerShell หรือ Terminal แล้วพิมพ์คำสั่งสร้างคู่กุญแจ (SSH Key Pairs) โดยใช้ Ed25519:
   ```bash
   ssh-keygen -t ed25519 -C "Miler-VPS"
   ```
   *(กด Enter ไปเรื่อยๆ เพื่อตกลงบันทึกไว้ในโฟลเดอร์เริ่มต้นของระบบ)*

2. **[บนคอมพิวเตอร์ส่วนตัว]** เปิดดูและคัดลอกรหัส Public Key ที่เราเพิ่งเจนเนอเรตเสร็จ:
   * **สำหรับ macOS:** คัดลอกลง Clipboard ทันทีด้วยคำสั่ง:
     ```bash
     pbcopy < ~/.ssh/id_ed25519.pub
     ```
   * **สำหรับ Windows (PowerShell) / ทั่วไป:**
     ```bash
     cat ~/.ssh/id_ed25519.pub
     ```
     *(คัดลอกรหัสที่แสดงขึ้นมาเก็บไว้)*

3. **[บนคอมพิวเตอร์ส่วนตัว]** ทดลองรีโมทล็อกอินเข้าไปที่ VPS ด้วยชื่อผู้ใช้ใหม่ที่เราสร้างจาก Step 2 (พิมพ์พาสเวิร์ดที่ตั้งไว้):
   ```bash
   ssh miler@<IP_Serverของคุณ>
   ```

4. **[บน Server (เมื่อรีโมทเข้ามาแล้ว)]** สร้างโฟลเดอร์สำหรับเก็บ SSH Key:
   ```bash
   mkdir -p ~/.ssh
   ```

5. **[บน Server]** สร้างและเปิดไฟล์ `authorized_keys` ด้วย text editor (Nano):
   ```bash
   nano ~/.ssh/authorized_keys
   ```

6. **[บน Server]** วาง Public Key ที่ Copy มาลงไปในไฟล์นี้ จากนั้นกดบันทึกโดยการกด:
   * `Ctrl + O` แล้วตามด้วย `Enter` (เพื่อ Save)
   * `Ctrl + X` (เพื่อออก)

---

#### ขั้นตอนการทดสอบ (ทำทั้งสองทางเลือก)
**[บนคอมพิวเตอร์ส่วนตัว]** ออกจาก SSH เดิม (`exit`) และลองรีโมทใหม่อีกครั้งเพื่อทดสอบว่าไม่ต้องใช้รหัสผ่านแล้วใช่หรือไม่:
```bash
ssh miler@<IP_Serverของคุณ>
```
*(หากสามารถเข้าใช้งานได้ทันทีโดยไม่ต้องใส่พาสเวิร์ด แสดงว่ากุญแจล็อกอินสำเร็จแล้ว!)*

### Step 4: ปิด Root Login และ Password Login (SSH Hardening)
ขั้นตอนนี้เพื่อระงับการล็อกอินด้วยรหัสผ่าน และระงับการล็อกอินโดยตรงจาก root ทุกกรณี บังคับล็อกอินผ่าน SSH Key ของคุณเท่านั้น
* **ทำที่ไหน:** บน Server (เข้าใช้งานด้วยสิทธิ์ผู้ใช้ใหม่ที่คุณสร้าง)

```bash
# 1. สำรองไฟล์ตั้งค่าเดิมเก็บไว้ก่อนกันพลาด
sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak

# 2. สร้างไฟล์การตั้งค่า Hardening ขึ้นมาใหม่ในโฟลเดอร์ย่อย (.d)
sudo nano /etc/ssh/sshd_config.d/hardening.conf
```

เมื่อเข้ามาในหน้า Nano editor ให้คัดลอกชุด Config ด้านล่างนี้ไปวาง:
```text
PermitRootLogin no
PasswordAuthentication no
KbdInteractiveAuthentication no
PubkeyAuthentication yes
PermitEmptyPasswords no
X11Forwarding no
```
*(กด `Ctrl + O` -> `Enter` เพื่อเซฟ และกด `Ctrl + X` เพื่อออกจากหน้าจอ)*

```bash
# 3. ตรวจสอบ Syntax ความถูกต้องของการตั้งค่า SSH
sudo sshd -t
```
*(หากกดแล้วไม่มีข้อมูลอะไรแสดงขึ้นมา ถือว่าการตั้งค่าถูกต้องไม่มีข้อผิดพลาดทางไวยากรณ์)*

```bash
# 4. สั่งให้ SSH โหลดชุด Config ใหม่ไปใช้
sudo systemctl reload ssh
```

### Step 5: เปิดใช้งาน Firewall ด้วย UFW
ควบคุมพอร์ต (Port) ของเซิร์ฟเวอร์ บล็อกการเข้าถึงจากภายนอกทุกช่องทาง ยกเว้นพอร์ตเฉพาะที่อนุญาตเท่านั้น
* **ทำที่ไหน:** บน Server

```bash
# 1. ตั้งนโยบายพื้นฐาน (บล็อกทุกขาเข้า / อนุญาตขาออก)
sudo ufw default deny incoming
sudo ufw default allow outgoing

# 2. อนุญาตพอร์ต SSH เสมอเพื่อไม่ให้ตัวเองโดนเตะออกจากเซิร์ฟเวอร์
sudo ufw allow OpenSSH

# 3. อนุญาตพอร์ตมาตรฐานของเว็บไซต์ HTTP (80) และ HTTPS (443)
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 4. เปิดใช้งาน UFW (พิมพ์ "y" ยืนยัน)
sudo ufw enable

# 5. ตรวจสอบสถานะการทำงานของ Firewall
sudo ufw status verbose
```

### Step 6: ติดตั้งและตั้งค่า Fail2Ban (ป้องกัน Brute Force)
ระงับ IP แปลกปลอมชั่วคราวกรณีที่ตรวจพบการพยายามสุ่มรหัสผ่านล็อกอินผิดพลาดซ้ำๆ
* **ทำที่ไหน:** บน Server

```bash
# 1. สร้างและแก้ไขไฟล์ jail สำหรับคุมพอร์ต SSH
sudo nano /etc/fail2ban/jail.d/sshd.local
```

คัดลอกค่า Config ต่อไปนี้ลงไป:
```ini
[sshd]
enabled = true
backend = systemd
port = ssh
maxretry = 5
findtime = 10m
bantime = 1h
```
> [!NOTE]  
> **ความหมาย:** คัดกรองตรวจพบการพยายามล็อกอินผิดซ้ำ 5 ครั้ง ภายในเวลา 10 นาที จะทำการแบน IP นั้นเป็นเวลา 1 ชั่วโมง  
> *(กด `Ctrl + O` -> `Enter` บันทึก และ `Ctrl + X` ออก)*

```bash
# 2. สั่งรันและตั้งให้เปิดบริการ Fail2Ban ทันทีเมื่อเปิดเครื่อง
sudo systemctl enable --now fail2ban

# 3. ตรวจสอบสถานะการล็อกของ jail
sudo fail2ban-client status sshd
```

---

### Step 7: เปิดใช้งานระบบ Automatic Security Update
ตัวแพ็กเกจ `unattended-upgrades` (ที่เราสั่งติดตั้งในสเต็ปแรก) จะอัปเดตแพตช์ความปลอดภัยของระบบให้โดยอัตโนมัติ

**วิธีการตรวจสอบสถานะ:**
```bash
# ตรวจเช็ค Log ว่าระบบทำงานหรือไม่
ls /var/log/unattended-upgrades/
```

## 🔍 คำสั่งเสริมที่มีประโยชน์สำหรับการตรวจสอบ (Useful Commands)

| คำสั่ง | คำอธิบาย |
| :--- | :--- |
| `sudo ss -tulpn` | ตรวจสอบพอร์ต (Port) ที่เปิดใช้งานบนระบบปัจจุบัน *(ควรเปิดเฉพาะพอร์ตที่จำเป็น เช่น 22, 80, 443 พอร์ตฐานข้อมูลอย่าง 3306, 5432, 6379 ไม่ควรถูกเปิดให้สาธารณะเข้าถึงได้โดยตรง)* |
| `sudo journalctl -u ssh --since today` | ดู Log การเชื่อมต่อ SSH ของวันนี้ |
| `last -a \| head` | ดูประวัติการเข้าสู่ระบบล่าสุด |
| `sudo fail2ban-client status sshd` | ดูรายชื่อ IP ที่ถูกบล็อกโดย Fail2Ban ณ ปัจจุบัน |

## 📌 สรุปกฎเหล็ก 5 อย่าสำหรับ Dev

1. **อย่า** ใช้ `root` ทำงานเป็นประจำ (ให้ใช้ User ทั่วไปแล้วใส่ `sudo` แทน)
2. **อย่า** เปิดพอร์ตทั้งหมดแบบสาธารณะ (เปิดเฉพาะที่จำเป็นและใช้ Reverse Proxy ซ่อนแอปข้างหลัง)
3. **อย่า** ใช้ Password ล็อกอินถ้าสามารถใช้ SSH Key ได้
4. **อย่า** ละเลยการรันระบบ Security Updates ของเซิร์ฟเวอร์
5. **อย่า** ลืมทำ Backup/Snapshot ทุกครั้งก่อนแก้ไข Config ใหญ่ๆ
